Caso 1

Tenemos una base de datos simple, que provee información a un solo usuario, por lo que el crecimiento de los requerimientos y de la base de datos es limitado. Entonces tal vez pasar de la 3FN sería algo exagerado, o tal vez llegar solo a una forma normal si el modelo no es complejo.

Caso 2

Tenemos una base de datos compleja, con miles de usuarios a los cuales proveer y el crecimiento es muy rápido. Para este caso sería obligatorio como mínimo llegar a una 3FN y explorar las otras formas normales para eliminar dependencias múltiples y claves relacionadas.
